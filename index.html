Windows Clipboard Viewer

Inspect all clipboard contents that the browser can access — including text, HTML, images, and files — and see how each maps to standard Windows CF_* clipboard format identifiers.

How this works: The browser Clipboard API exposes data as MIME-typed blobs, not raw Windows clipboard handles. The table below maps what the browser can read to the corresponding Windows CF_* format — and identifies which formats are permanently blocked by the browser sandbox.

  • CF_TEXT / CF_UNICODETEXT — available as text/plain
  • HTML Format — available as text/html
  • CF_BITMAP / CF_DIB — available as image/png (browser converts)
  • CF_HDROP — partially available via paste event (file names and sizes; not full paths)
  • CF_ENHMETAFILE, CF_METAFILEPICT, CF_LOCALE — not accessible from any browser page; require native Win32 access

Press Ctrl+V anywhere on the page to inspect your clipboard.
